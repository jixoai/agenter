### Chat Open, Scrolled Up

Expected:

- `assets/cli-shell-chat-tui-reference-v9-chat-right-scrolled-grid.png`
- `assets/cli-shell-chat-tui-reference-v9-chat-right-scrolled-grid.txt`

Actual implementation evidence:

- `packages/cli-shell/test/cli-shell-tui.test.ts`
  - `Scenario: Given Chat is scrolled up When a user message is sent Then cli-shell returns the transcript to bottom-pinned mode`
  - `Scenario: Given dialogue backend owns scroll When the wheel targets dialogue Then shell viewport is not changed by the event`
  - `Scenario: Given Chat placement toolbar is visible When cover control is clicked Then the requested placement becomes cover`
- Local ignored render evidence:
  - `.chat/separate-cli-shell-product-from-terminal-view-components/chat-tui-v9-actual-chat-right-scrolled.png`
  - `.chat/separate-cli-shell-product-from-terminal-view-components/chat-tui-v9-actual-chat-right-scrolled.txt`

Observed:

- Chat viewport can be deliberately offset from bottom.
- Scrollbar thumb is not at the bottom while scrolled up.
- Compact stick-to-bottom button is visible when `dialogueScrollOffset > 0`.
- Newer content can exist below the current view without forcing the transcript back to bottom.

Failure signal:

- User scroll-up is clamped back to bottom while new content exists.
- Stick-to-bottom action is absent when `dialogueScrollOffset > 0`.
- Stick-to-bottom action does not reset `dialogueScrollOffset` to `0`.

## Code Evidence Mapping

- `packages/cli-shell/src/tui/model.ts`
  - Bottom toolbar no longer appends the old `terminal` label or visible shortcut text.
  - Smart placement may return `cover` when side/floating placements do not fit.
- `packages/cli-shell/src/tui/frame.ts`
  - `cover` placement consumes the full body area and suppresses shell scrollbar regions while covered.
  - Toolbar click regions still map managed and Chat entry actions.
- `packages/cli-shell/src/tui/dialogue-surface.ts`
  - Chat panel is frameless and background-driven.
  - Message list always renders a scrollbar column.
  - Scrolled-up state renders `stickDialogueToBottom`.
  - `▾` maps to explicit `placeCover` instead of duplicating floating placement.
- `packages/cli-shell/src/tui/controller.ts`
  - Chat paste appends to `dialogueDraft` instead of being discarded.
  - Successful send clears the draft and resets `dialogueScrollOffset` to `0`.
  - `stickDialogueToBottom` and `placeCover` are explicit product actions.
- `packages/client-sdk/src/product-extension-runtime.ts`
- `packages/terminal-system/src/terminal-control-plane.types.ts`
- `packages/terminal-system/src/terminal-control-plane.ts`
- `packages/app-server/src/trpc/router.ts`
  - `dialoguePlacement` schema and terminal-2 metadata preserve `cover`.
