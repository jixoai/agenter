# ChatTUI v9 中文人工验收任务

这份清单给用户做最终 `cli-shell` 集成验收。简单单元测试能覆盖的内容不用人工重复；这里专门覆盖真实终端体验。

## 1. 底部状态栏

1. 启动 `agenter shell`。
2. 确认默认界面只有底部一行工具栏，没有顶部状态栏、侧边栏、tab、dashboard。
3. 底部左侧应是状态图标。
4. 状态图标右侧应是当前活动的流式片段。
5. 底部右侧应只有托管开关和 Chat 入口，Chat 入口显示未读数量。
6. 底部不应显示 `Heartbeat` 或 `心跳` 文字。
7. 底部不应显示快捷键说明。

通过标准：肉眼看到的效果接近 `assets/cli-shell-chat-tui-reference-v9-toolbar-grid.png`。

## 2. Chat 面板打开与关闭

1. 点击底部 Chat 入口。
2. 再次点击底部 Chat 入口，确认可以关闭 Chat 面板。
3. 打开后也点击 Chat 面板右上角关闭按钮，确认可以关闭。
4. 打开 Chat 面板时，Shell 主体仍然可识别，不应变成 Web 卡片风格。

通过标准：入口本身既能打开也能关闭；右上角关闭按钮保留。

## 3. Chat 面板视觉

1. 打开 Chat 面板。
2. 确认面板顶部有工具栏，左侧是布局按钮，右侧是关闭按钮。
3. 确认消息列表右侧有滚动条。
4. 确认用户消息有灰色背景，并且左侧留白处有 `>` 标记。
5. 确认底部输入框有一条上边界/分割线，背景偏灰，左侧留白处有 `>` 标记。
6. 确认输入框默认聚焦并显示光标。

通过标准：肉眼看到的效果接近 `assets/cli-shell-chat-tui-reference-v9-chat-right-pinned-grid.png`。

## 4. 传统聊天室贴底行为

1. 打开 Chat 面板并滚动到最底部。
2. 让助手输出新消息或流式 message-part。
3. 确认列表自动跟随到底部。
4. 手动向上滚动消息列表。
5. 再让助手输出新消息或流式 message-part。
6. 确认列表不会被强制拉回底部。
7. 确认非贴底状态下出现 `↓` 或等价的贴底/新消息按钮。
8. 点击该按钮。
9. 确认列表回到底部，按钮消失。
10. 再次向上滚动后，在输入框发送一条自己的消息。
11. 确认发送成功后列表回到底部，并继续自动跟随后续输出。

通过标准：肉眼看到的非贴底效果接近 `assets/cli-shell-chat-tui-reference-v9-chat-right-scrolled-grid.png`。

## 5. Shell 与 Chat 输入隔离

1. Shell 聚焦时输入普通命令，确认输入进入 Shell。
2. Chat 输入框聚焦时输入多行内容，确认输入进入 Chat。
3. Chat 输入框聚焦时粘贴内容，确认不会进入 Shell。
4. Shell 聚焦时粘贴命令，确认不会进入 Chat。
5. 在 Chat 面板中滚动消息列表，确认滚动的是 Chat 消息，不是 Shell 视口。
6. 在 Shell 中滚动，确认滚动的是 Shell 视口，不是 Chat 消息列表。

通过标准：Shell 和 Chat 是两个独立输入/滚动世界，事件按聚焦 Box 和相对坐标路由。

## 6. 受限空间覆盖模式

1. 缩小终端窗口，让右侧/左侧 docked Chat 放不下。
2. 打开 Chat 面板。
3. 确认 Chat 可以切换为覆盖式完整面板。
4. 确认覆盖式面板仍保留顶部工具栏、消息滚动条、底部输入框和贴底按钮行为。
5. 关闭 Chat 后，确认返回一行底部状态栏和 Shell 主体。

通过标准：空间不足时不出现破碎布局；覆盖模式可以接受。

## 7. 最终判定

如果以下任一现象出现，应判定失败：

- 底部显示 `Heartbeat` / `心跳` 文字。
- 底部显示快捷键说明。
- Chat 消息列表没有滚动条。
- 非贴底时没有贴底/新消息按钮。
- 用户向上滚动后，新消息强制抢回到底部。
- 发送自己的消息后 Chat 自动关闭。
- Chat 聚焦时输入或粘贴进入 Shell。
- Shell 聚焦时输入或粘贴进入 Chat。
- Chat 面板是带完整粗边框的 Web 卡片风格，而不是终端 frameless 面板。
