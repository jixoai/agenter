# Walkthrough Evidence

Each walkthrough in this change should be written as one Markdown file in this directory.

Minimum structure:

## Scenario
- Name of the walkthrough case

## Commands
- Exact commands, API calls, or scripted steps used

## Expected
- The durable contract that should hold

## Actual
- What actually happened

## Evidence
- Log paths
- DB queries
- request/response excerpts
- screenshots or exported artifacts when relevant

## Verdict
- pass / fail
- brief explanation
